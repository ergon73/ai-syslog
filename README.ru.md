# ai-syslog

**Syslog-сервер с ИИ-разбором: каждая новая ошибка в журнале роутера мгновенно получает объяснение от LLM — причина, серьёзность и что делать.**

[English version](README.md)

## Идея

Роутер генерирует непрерывный поток syslog-сообщений. 95% — повторяющийся шум, остальное — криптичные однострочники вроде `curl response code: 403, content length: 17`, для расшифровки которых нужна экспертиза. ai-syslog принимает весь поток, хранит его целиком, а LLM объясняет **только новое и только важное** — экспертные аннотации обходятся дешевле $1/месяц.

## Как это устроено

```
роутер ──UDP 514──▶ листенер ──▶ SQLite (сырой лог, весь и всегда)
                                    │
                              Drain3: выделение шаблонов
                                    │  только НОВЫЙ шаблон + severity ≤ warning
                                    ▼
                              LLM-триаж (structured output: серьёзность,
                              причина, рекомендация, уверенность)
                                    │
                                    ▼
                              FastAPI-дашборд: живой лог с аннотациями
                              inline + суточный ИИ-дайджест
```

### Экономика — в основе дизайна

1. **Шаблоны (Drain3)**: 150 одинаковых ошибок `403` схлопываются в один кластер.
2. **Порог серьёзности**: в LLM уходит только warning и хуже — DHCP-шум не тратит токены.
3. **Аннотация один раз — показ везде**: разбор привязан к шаблону и отображается у всех совпадающих строк бесплатно.
4. **Два уровня моделей**: дешёвая для триажа (≈$0.0003/вызов), сильная для дайджеста (≈$0.005/вызов).

Реальная стоимость для домашнего роутера: **меньше $0.50/месяц**.

## Быстрый старт

```bash
git clone https://github.com/ergon73/ai-syslog && cd ai-syslog
python -m venv .venv && .venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env       # вписать OPENROUTER_API_KEY
python main.py
```

Дашборд: http://127.0.0.1:8514. Без API-ключа сервер тоже работает: принимает лог и считает шаблоны, LLM-разбор выключен.

### Настройка роутера (пример: Keenetic)

Веб-интерфейс: Управление → Диагностика → «Системный журнал» → отправка на удалённый syslog-сервер → IP машины с ai-syslog. Или CLI:

```
system log server <IP-коллектора>
system configuration save
```

Подойдёт любое устройство с syslog по RFC 3164/UDP. На Windows откройте входящий UDP 514 от роутера:

```powershell
New-NetFirewallRule -DisplayName "ai-syslog UDP 514" -Direction Inbound `
  -Protocol UDP -LocalPort 514 -RemoteAddress <IP-роутера> -Action Allow
```

## Конфигурация

Всё через `.env` (см. [.env.example](.env.example)): ключ провайдера (OpenRouter приоритетнее Anthropic), модели триажа/синтеза, `DEVICE_PROFILE` — описание вашей сети для системного промпта (чем конкретнее, тем точнее разборы), порог серьёзности и порты.
